# AI Agent Template Repository

**Purpose**: このリポジトリは、AIエージェント（Claude Code等）が効果的に作業するためのテンプレートと構成を提供します。

## 概要

このリポジトリは、AIエージェントによるソフトウェア開発の効率化を目的とした**テンプレート**です。

- ✅ AIエージェントが即座に作業を開始できる環境
- ✅ 一貫性のある開発プロセスの実現
- ✅ AIエージェントの行動方針を明確化
- ✅ プロジェクト全体の構造化

## Quick Start - 5ステップで始めよう

このテンプレートをあなたのプロジェクトですぐに使うための最短ルート：

### 1. テンプレートの複製

```bash
git clone <template-url> <your-project-name>
cd <your-project-name>
```

### 2. プロジェクト目的を定義

`PURPOSE.md` を編集して、あなたのプロジェクトの目的を定義：

```markdown
# プロジェクト名

## 大目標
このプロジェクトが実現する価値を1行で

## 今期の優先順位
1. 最も重要な milestone
2. 次に重要な milestone
```

### 3. システムルールの設定

`SYSTEM_CONSTITUTION.md` を編集して、絶対ルールを定義：

```markdown
# 禁止事項
- × 絶対にやってはいけないこと

# 必須事項
- ○ 必ず守るべきこと
```

### 4. Concept Syncの初期化（オプション）

AIエージェントと協業する場合、Concept Syncを有効にします：

```bash
# .concept/charter.yml が自動生成されます
# AIエージェントは charter.yml を見てプロジェクトのゴールを理解します
```

### 5. 開発開始

AIエージェント（Claude Code等）を起動して、最初のタスクを依頼：

```bash
claude  # またはあなたのAIツール
```

```

"PURPOSE.mdに書いた milestones の最初のものを実行して"
```

**これだけで準備完了！** 🚀

## セットアップ手順（詳細）

このテンプレートを使用するには：

### 1. リポジトリの複製

```bash
git clone <template-url> <your-new-repo>
cd <your-new-repo>
```

### 2. プロジェクト設定のカスタマイズ

#### 2.1. PURPOSE.md の編集

プロジェクトの目的、大目標、今期の優先順位を定義します：

```markdown
# プロジェクト名

## 大目標
このプロジェクトが実現する価値を1行で記述

## 現在のステータス
- フェーズ: [MVP開発 / 機能拡張 / 保守]
- 開始日: YYYY-MM-DD

## 今期の優先順位
### P0 - 最優先（今週完了）
1. [最も重要なマイルストーン]
   - 受入基準: [具体的な指標]
   - 期限: [日付]

### P1 - 高優先（今月完了）
1. [次に重要なマイルストーン]
```

#### 2.2. SYSTEM_CONSTITUTION.md の編集

システムの絶対ルールを定義します：

```markdown
# システム憲法

## 禁止事項（絶対遵守）
- 生産環境DBへの直接接続・変更
- 秘密情報のリポジトリへのコミット
- main/masterブランチへの直接push

## 必須事項
- 全てのコード変更はPRを通す
- CIテストが全てパスすること
- コードレビューを1人以上受ける
```

#### 2.3. .claude/settings.json の確認

Claude Codeの基本設定を確認します（必要に応じてカスタマイズ）：

```json
{
  "alwaysAllowWrite": false,
  "alwaysAllowMarketplace": false
}
```

### 3. Concept Sync の初期化（推奨）

AIエージェントと協業する場合、Concept Syncを有効にします：

```bash
# AIエージェントは自動的に .concept/ ディレクトリを初期化します
# charter.yml が生成され、プロジェクトのゴール管理が開始されます
```

Concept Syncにより以下が管理されます：
- **charter.yml**: ゴール・マイルストーン・成功指標
- **ontology.yml**: プロジェクト固有の概念辞書
- **invariants.yml**: 変更不可のルール
- **decisions.md**: 設計判断の記録

### 4. 最初のタスクを依頼

AIエージェント（Claude Code等）を起動：

```bash
claude
```

最初のタスクを依頼：

```
"PURPOSE.mdのP0マイルストーンを実行して"
```

**これだけで、AIエージェントが自律的に開発を開始します！** 🚀

## ディレクトリ構造

```
.claude/
├── skills/           # AIエージェント向けのカスタムスキル定義
├── agents/           # 専門AIエージェント定義
├── commands/         # カスタムコマンド定義
└── hooks/            # Claude Code フック

.concept/             # コンセプト管理（Source of Truth）
├── charter.yml       # プロジェクト憲章
├── autopilot.yml     # 自動化設定
└── ontology.yml      # オントロジー

AGENTS.md             # AIエージェントへの指示
CLAUDE.md             # Claude Code に関する指示
PURPOSE.md            # プロジェクトの目的
SYSTEM_CONSTITUTION.md # システムの基本ルール
```

## 主な機能

### 1. カスタムスキル（`.claude/skills/`）

AIエージェント向けのカスタムスキル定義：

- **commit-prep-helper**: コミット準備・Lint・テスト実行
- **stepwise-executor**: 段階的タスク実行・作業管理
- **codebase-improvement-advisor**: プロジェクト改善提案
- **spec-flow-auto**: 仕様書・フロー自動生成
- その他多数

### 2. 専門エージェント（`.claude/agents/`）

高度な専門知識を持つAIエージェント：

- **git-expert**: Git操作の専門家
- **refactoring-expert**: リファクタリングの専門家
- **code-review-expert**: コードレビューの専門家
- **triage-expert**: トラブルシューティングの専門家
- その他多数の専門エージェント

### 3. カスタムコマンド（`.claude/commands/`）

よく使う操作をカスタムコマンド化：

- `/git:status` - Gitステータスの確認
- `/code-review` - コードレビューの実行
- `/test` - テストの実行
- `/verify` - システム検証
- `/deploy` - デプロイの実行
- その他多数のカスタムコマンド

### 4. BMAD フレームワーク

ビジネス要件から実装までのBMADベースの開発ワークフロー：

- 仕様書（PRD・Architecture・Epics & Stories）
- 開発計画
- 自動テスト
- 文書生成

## 使用例

### 例1: 新機能の開発

AIエージェントへの指示例：

```
"ユーザー認証機能を実装して。PURPOSE.mdのP0-1にある通り"
```

AIエージェントは以下を自律的に実行します：
1. PURPOSE.mdからゴールを確認
2. 現在のコードベースを分析
3. 実装計画を立案
4. コードを実装
5. テストを追加
6. コードレビューを依頼

### 例2: バグ修正

AIエージェントへの指示例：

```
"ログイン時のエラーを調査して修正して"
```

AIエージェントは以下を実行します：
1. エラーログを調査
2. 再現手順を確認
3. 原因を特定
4. 修正を実装
5. 回帰テストを実行
6. 再発防止策を提案

### 例3: リファクタリング

AIエージェントへの指示例：

```
"ユーザー管理モジュールをリファクタリングして。可読性を向上させて"
```

AIエージェントは以下を実行します：
1. 現在のコードを分析
2. 改善点を特定
3. リファクタリング計画を立案
4. 段階的にリファクタリングを実施
5. 全テストがパスすることを確認
6. 変更内容を要約

### 例4: ドキュメント生成

AIエージェントへの指示例：

```
"APIドキュメントを自動生成して"
```

AIエージェントは以下を実行します：
1. コードからAPI定義を抽出
2. OpenAPI/Swagger形式でドキュメント生成
3. 使用例を追加
4. ドキュメントをdocs/以下に配置
